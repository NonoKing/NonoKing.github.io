解决Github访问速度慢的问题

发表于 2020-10-15 更新于 2020-10-20 分类于 GitHub
本文字数： 430字

背景

在使用GitHub的使用，发现无论是访问官网，还是用git clone下载repo，速度都慢的要死

网上有很多方法可以参考，例如这篇文章，例举了很多方法

方法

我使用的是修改host文件的方法

环境：win10 64位

首先访问ip138
查找 github.com 和 github.global.ssl.fastly.net 的ip地址，并复制下来
用文本编辑软件（比如记事本）打开 hosts文件

比如win10 64位，系统盘是c盘的
打开C:/windows/system32/drivers/etc/hosts
在里面添加

  1. 52.74.223.119 github.com
  2. 108.160.165.53 github.global.ssl.fastly.net

(你查到的ip地址，可能跟我在上面列举的不完全一致，可能会变化)
保存之后，打开cmd，输入ipconfig /flushdns回车即可生效

然后你就会惊喜的发现无论是访问官网，还是git clone，git push速度都比之前快了很多
